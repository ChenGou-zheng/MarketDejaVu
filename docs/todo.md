### 动态策略配置系统改进与实现建议书（Agent 执行 TODO List）

#### 🎯 总体目标
基于“相似行情→策略表现”的前瞻统计推断，构建一个没有未来信息泄露、能真实判断历史策略有效性的动态配置系统，从根本上避免回顾式排名和直接预测模型带来的巨大失效。

---

### 1️⃣ 问题诊断与数据准备
- [ ] **1.1** 梳理并明确两次失败根因：
  - HMM 状态回顾式排名：使用了全期或训练集未来信息，单状态样本少且极端收益主导，无法外推。
  - XGBoost 直接预测策略收益：在极低信噪比环境中学习，AUC≈0.6 几乎无实用价值，本质噪声建模。
- [ ] **1.2** 确定策略池：
  - 为每个策略编写可执行的收益计算函数，确保能获取历史每日净值。
- [ ] **1.3** 定义基准：
  - 基准定为沪深300全收益指数（或对应 ETF 模拟收益）。
- [ ] **1.4** 明确决策与反馈周期：
  - 设定向前看周期 N = 20 个交易日。
  - 每日生成一个决策时点 t，根据 t 之前信息选择策略，持有 N 天后评估表现。

---

### 2️⃣ 构建完全无未来信息的特征库与片段库
- [ ] **2.1** 确定市场特征向量 F(t) 维度（所有计算必须在 t 时点可用）：
  - 技术面：过去20/60日动量、均线排列（如 MA5>MA20）、RSI。
  - 波动率：20日历史波动率分位数、ATR 占比。
  - 回撤：滚动60日最大回撤、回撤恢复天数。
  - 收益率：近期偏度、涨跌比率。
  - 宏观/政策：Shibor/国债利率、社融增速（滞后公布需取已知值）、政策事件哑变量（如前推规则避开未来事件）。
- [ ] **2.2** 构建历史特征序列：
  - 在每一个历史交易日 t，依据截至当天收盘数据计算 F(t)。
  - 形成 `(t, F(t))` 表，确保从最早可用日期开始，不引入任何 t 之后信息。
- [ ] **2.3** 构建策略收益快照库：
  - 对于每个策略 S 和每个历史时点 t，记录如果从 t 收盘买入并持有 N 天的超额收益（相对基准）。
  - 该收益序列完全基于未来实际走势，但仅在历史回测评估时使用，**严禁用于筛选策略的参数训练**。

---

### 3️⃣ 实现前瞻性相似度匹配框架（实时模拟实验）
- [ ] **3.1** 标准化特征：计算 F(t) 的滚动均值和标准差（仅使用 t 之前数据），对每个时点进行 z-score 标准化。
- [ ] **3.2** 定义相似度度量：
  - 采用马氏距离或加权欧氏距离（宏观变量可适当提高权重）。
  - 对久远的历史样本施加时间衰减因子（如半衰期3年）。
- [ ] **3.3** 滚动模拟“实时决策实验”：
  - 从最早允许的决策时点 T_min 开始，直到最新样本外时点。
  - 对每一个 t：
    - **只使用** 所有 τ < t 的历史片段。
    - 计算 F(t) 与所有历史 F(τ) 的相似度距离。
    - 选择距离最小的 K 个最相似历史时刻（K 初始设为 30，后续可根据显著性样本量调整）。
    - 提取这 K 个时刻后续 N 天的各策略超额收益序列（此时这些收益对于 t 时刻已是“历史”，因 τ+N < t 才可用，需要额外保证：只选那些 τ+N < t 的相似时刻，避免信息泄露！）。务必检查 `τ + N < t` 条件，若 K 不足则扩大池，并记录实际可用样本数。

---

### 4️⃣ 前瞻统计检验与策略真实有效性判断
- [ ] **4.1** 对每个策略 S 在 t 时刻，基于相似历史样本计算：
  - 平均超额收益 μ。
  - 超额收益的标准误 SE。
  - t 统计量 = μ / SE。
  - 单边 p 值 (H1: μ > 0 即显著跑赢基准)。
  - 胜率（超额 > 0 的比例）和盈亏比。
- [ ] **4.2** 多重比较校正（关键）：
  - 因同时检验多个策略，必须控制假阳性。
  - 应用 Benjamini-Hochberg FDR 控制（q=0.1）或 Bonferroni 校正。
  - 只当策略 S 的调整后 p 值 < 设定阈值（如 0.1）时，才标记为“真实良好”。
- [ ] **4.3** 生成每个时点 t 的“有效策略候选集”：
  - 包含策略名称、统计显著性指标、置信度、可用相似样本数、平均超额等。
  - 若无任何策略通过检验，则当日不调用任何策略（保持空仓或跟踪基准）。

---

### 5️⃣ 回测整个动态选择系统的真实绩效
- [ ] **5.1** 基于上述每日决策实验的结果，构建动态策略组合：
  - 对于 t 日选出的有效策略，采用置信度加权或等权分配仓位（需扣除交易手续费，单边至少千分之一）。
  - 持有 N 天后平仓并重新决策，或每日再平衡（视设计而定）。
- [ ] **5.2** 计算组合扣费后净值曲线与基准对比：
  - 年化超额收益、跟踪误差、信息比率。
  - 最大回撤、胜率、盈亏比。
- [ ] **5.3** 对整体决策系统进行显著性检验：
  - 使用 **Deflated Sharpe Ratio (DSR)** 检验扣费后的超额夏普是否显著（考虑多次试验和数据挖掘）。
  - 若 DSR 的 p 值 > 0.05，则系统整体不真实，需返回调整特征或参数。
- [ ] **5.4** 进行样本外测试：
  - 固定模型和参数后，在全新时间区段（如2024-2025）运行完全独立的样本外验证，严禁任何参数调整。

---

### 6️⃣ 改造现有 HMM 与 XGBoost 尝试
- [ ] **6.1 在线 HMM 方案（改造尝试1）：**
  - 改用滚动窗口 HMM：每个交易日 t，仅使用 t 之前数据训练 HMM，得到该日状态标签（隐藏状态序列实时确定，不进行未来平滑）。
  - 记录每个历史状态出现后各策略 N 日超额收益（同样需满足事件已发生，即状态出现时点 τ + N < t）。
  - 在新一日状态确认后，调取该状态所有历史后续收益，执行与步骤 4 完全相同的统计检验，显著才纳入。
- [ ] **6.2 XGBoost 元模型方案（改造尝试2）：**
  - 不再直接预测策略收益，改为构建二层模型。
  - 特征工程：使用步骤 4 中为每个策略计算出的统计量作为特征，如“相似历史平均超额”、“相似历史t值”、“胜率”、“样本量”、“平均相似距离”等。
  - 标签：该策略在未来 N 天是否显著跑赢基准（可设超额 > 2% 为1，否则0）。
  - 训练时必须使用严格时序交叉验证（Purged Walk-Forward），严禁随机拆分。
  - 评估该元模型的 AUC 和实际调用后的组合绩效，只有显著优于简单统计筛选才采用。

---

### 7️⃣ 关键防坑检查与执行保障
- [ ] **7.1 信息泄露检查工具：**
  - 编写脚本验证在所有特征计算、相似样本提取、统计检验步骤中，绝不使用 t 时点之后的数据。可打时间戳自动化检查。
- [ ] **7.2 交易成本硬约束：**
  - 所有收益计算默认扣除双边 0.1% 滑点与手续费，高频类信号需扣更多。
- [ ] **7.3 策略池生存偏差清理：**
  - 只纳入有连续可投资标的的指数增强或行业轮动策略，剔除退市股组合。
- [ ] **7.4 相似度计算中的宏观权重校验：**
  - 定期做敏感度分析，如果仅用技术面相似度，选出的策略超额是否显著变差，以确定宏观因子的必要性。
- [ ] **7.5 系统输出格式：**
  - 最终交付物包括：每日策略调用信号、置信度及依据（相似行情图、统计检验结果），用于人工核验。

---

> **文档说明**：本 TODO list 按执行顺序排列，必须严格遵循“绝对无未来信息”的准则。每完成一个阶段**应进行单元测试**和泄露检查，确认无误后再推进下一阶段。最终系统需在样本外持续监控，若 DSR 或超额显著性消失，应立即排查实现是否隐含泄露。open
